# User-material intake

Use this process whenever the user supplies attachments, local file or folder paths, or authorized URLs.

## Supported inputs

- Attached or local PDF, DOCX, PPTX, XLSX, CSV, TXT, Markdown, and image files.
- A folder containing multiple project files.
- User-authorized webpages or connected documents.

Read files in place. Do not copy raw source materials into the output folder unless the user explicitly requests it.

## Intake sequence

1. Inventory filenames, paths/URLs, formats, document dates, apparent versions, and duplicates.
2. Ask only blocking questions: passwords, unreadable scans, unclear confidentiality, or ambiguous final/draft versions.
3. Assign stable IDs `U001`, `U002`, and so on.
4. Record each item in the `user_materials` worksheet.
5. Extract a concise summary, usable claims, quantitative data, definitions, geography, time period, and limitations.
6. Classify evidence status as `Internal fact`, `Client assertion`, `Draft`, `Historical`, or `Reference`; assign confidence separately.
7. Map usable evidence to the nine-part research matrix and to claim IDs.
8. Compare material-derived claims with current public evidence. Preserve and explain contradictions rather than silently choosing one.
9. State which materials were used, excluded, unreadable, superseded, or insufficient in the report methodology.

## Confidentiality and safe handling

- Treat user materials as private by default.
- Do not place confidential phrases or contents into web-search queries unless the user explicitly authorizes it.
- Do not upload, publish, or redistribute raw internal files.
- Quote minimally; redact personal data, credentials, customer lists, and commercially sensitive details when they are not essential.
- A user-provided statement is not automatically a verified fact. Keep client-only claims labeled and separate from independently corroborated findings.
- Respect any usage restriction stated by the user or inside the material.

## Conflicts and version control

When sources disagree, compare document date, market, definition, measurement method, and version. Record both positions, identify the better-supported interpretation, and mark unresolved points as a gap or verification item.

## Required traceability

- Public sources use `S...` IDs; user materials use `U...` IDs.
- Every material-derived factual claim must cite at least one U-ID.
- The claims ledger may combine S- and U-IDs when a conclusion is corroborated.
- The final report must contain a `User-provided materials and limitations` section.
